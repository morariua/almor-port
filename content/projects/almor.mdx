# Project Overview

This website is a modern, performant, and highly customizable platform built using cutting-edge technologies. Below is a detailed breakdown of the tech stack, features, and build process.

## Tech Stack

### Frontend
- **Next.js**: A React framework for server-side rendering, static site generation, and API routes.
- **Tailwind CSS**: A utility-first CSS framework for rapid UI development.
- **MDX**: Markdown with JSX support for dynamic and interactive content.
- **Lucide Icons**: A lightweight and customizable icon library.
- **Particles.js**: Used for creating interactive background animations.

### Backend
- **Upstash Redis**: A serverless Redis database for storing and retrieving page views and other dynamic data.
- **Contentlayer**: A content management tool for transforming Markdown/MDX files into type-safe JSON data.

### Deployment
- **Vercel**: A serverless platform for deploying Next.js applications with automatic CI/CD.

### Analytics
- **Custom Redis Integration**: Page views are tracked and stored in a Redis database, allowing for real-time analytics.

---

## Features

### Dynamic Project Pages
- Each project is dynamically rendered using the `[slug]` route in Next.js.
- Project data is sourced from JSON files, which are parsed and transformed into type-safe objects using Contentlayer.

### Responsive Design
- The website is fully responsive, ensuring a seamless experience across all devices.
- Tailwind CSS is used for styling, enabling rapid prototyping and consistent design.

### Interactive Components
- **Skills Drawer**: A collapsible drawer component for showcasing skills and technologies.
- **Particles Background**: An interactive background animation powered by Particles.js.

### SEO Optimization
- Metadata (title, description, etc.) is dynamically generated for each page, improving search engine visibility.

## Build Process

### Step 1: Project Setup
1. **Initialize Next.js**: The project was bootstrapped using `create-next-app`.
2. **Install Dependencies**: Added Tailwind CSS, Contentlayer, Upstash Redis, and other required libraries.

### Step 2: Content Management
1. **Create MDX/JSON Files**: Each project is represented as an MDX or JSON file in the `content/projects` folder.
2. **Contentlayer Integration**: Contentlayer is configured to parse these files and generate type-safe data for the website.

### Step 3: Dynamic Routing
1. **Slug-Based Routing**: The `[slug]` dynamic route in Next.js is used to render individual project pages.
2. **Data Fetching**: Project data is fetched from the generated JSON files and passed to the page components.

### Step 4: Styling and Components
1. **Tailwind CSS**: Utility classes are used to style components and ensure a consistent design.
2. **Reusable Components**: Components like `Card`, `Article`, and `Navigation` are created for reusability.

### Step 5: Analytics Integration
1. **Redis Setup**: Upstash Redis is configured to store and retrieve page views.
2. **View Tracking**: Page views are tracked using a custom API route and displayed on the project pages.

### Step 6: Deployment
1. **Vercel Integration**: The website is deployed on Vercel with automatic CI/CD.
2. **Environment Variables**: Sensitive data (e.g., Redis credentials) is stored in environment variables for security.

## Challenges and Solutions

### Challenge 1: Dynamic Content Generation
- **Problem**: Generating dynamic project pages from JSON/MDX files required a robust content management solution.
- **Solution**: Contentlayer was integrated to parse and transform content into type-safe data.

### Challenge 2: Real-Time Analytics
- **Problem**: Tracking page views in real-time required a fast and scalable database.
- **Solution**: Upstash Redis was used to store and retrieve page views efficiently.

### Challenge 3: Responsive Design
- **Problem**: Ensuring a consistent design across all devices was time-consuming.
- **Solution**: Tailwind CSS was used to streamline the styling process and ensure responsiveness.
